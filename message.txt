Всем привет
Описал что нам примерно нужно от бэка

1
Авторизация должна выдавать какой-то токен или id или что-то ещё чтобы потом этот токен юзер передавал в запросы

2
При создании нового пользователя помимо логина и пароля необходимо задавать его
name (строка - имя отображаемое другим пользователям, может содержать Имя/Имя Фамилия/никнейм и вообще всё что угодно)
phone (строка - контактный телефон, неплохо бы его валидировать, но на первых порах можно просто без проверок принимать всё что пользователь впишет)

3
Пользователь может запросить список всех идей которые есть у нас в базе (пока думаю можем не заморачиваться с пагинацией а просто выдавать всё что есть)
вот такая структура[
  {
    "id": "1",
    "title": "Idea 1",
    "description": "Bla-bla-bla big text",
    "likes": 10,
    "date": 1223232424,
    "isFavorite": true,
    "author": {
      "id": "1",
      "name": "Maria",
      "phone": "+1234566789"
    }
  },
  {
    "id": "2",
    "title": "Idea 2",
    "description": "Bla-bla-bla big text 2",
    "likes": 0,
    "date": 1223232424,
    "isFavorite": false,
    "author": {
      "id": "2",
      "name": "author123",
      "phone": "+2234566789"
    }
  }
]
title - строка название/заголовок
description - строка с длинным описанием
likes - целое число количество лайков
date - long timestamp дата создания идеи
isFavorite - добавлял ли данный пользователь эту идею в понравившиеся/избранные
author - объект с инфой об авторе, насчёт id автора не уверен но какой-то уникальный идентификатор не помешал бы

список должен быть отсортирован по количеству лайков от большего к меньшему, при одинаковом количестве лайков по дате более новые над более старыми

4
Пользователь должен иметь возможность добавить в избранное/убрать из избранного идею

в запрос нужно передавать идентификатор самого юзера и id идеи и сохранять это в базе, актуализируя при этом количество лайков

наверное проще сделать два отдельных метода один для «добавить в избранное» и один для «убрать из избранного», возможно можно как-то обойтись одним методом тут я не знаю как лучше

5
Пользователь должен иметь возможность отправить свою идею
В запрос он будет передавать вот эти значения
{
    "title": "Idea 2",
    "description": "Bla-bla-bla big text 2",
    "date": 1223232424
}

можем допустить что пользователь сразу сам лайкает свою идею и проставить сразу likes = 1 и isFavorite = true, id генерим на бэке.
Убрать свой лайк мы ему оставляем возможность, чтобы не заморачиваться с этим

Также необходимо каким-то образом понимать что идея принадлежит самому текущему пользователю. Для этого нужно либо в идею добавить некий флаг isMine либо во время авторизации нужно выдавать юзеру помимо некого токена ещё его собственный id, чтобы мы могли сравнить его с id автора идеи, как проще, так и надо сделать.

6
Неплохо бы сделать отдельные два запроса:
а) всех избранных идей (все идеи которые юзер лайкнул)
б) его собственных идей (те что он создал)

Если это требует много времени можно ограничиться запросом всех идей (пункт 3) а мы его уже на фронте сами отфильтруем

7
Так же было бы удобно чтобы автор мог удалить свою идею и отредактировать её (но это можно напоследок оставить, если и будем это делать то в последнюю очередь)

Но вот какой-то механизм удалить неправильно созданную или больше ненужную идею было бы неплохо иметь чисто в дебажных целях (чтобы этот метод не проверял пользователя и давал любую идею удалить по id)

8
Было бы удобно для юзера иметь возможность запросить свои юзер инфо (имя и телефон для отображения в неком экране профиля), также есть варик просто присылать их в ответе авторизации если так проще

И также было бы удобно иметь возможность изменить свои имя и телефон каким-то отдельным запросом, но это опять же в последнюю очередь.

Если есть какие-то вопросы/предложения задавайте и предлагайте, но думаю нужно всё стараться делать как можно проще, сдать проект, а всякие плюшки уже потом если понадобиться докручивать
